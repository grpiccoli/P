Estandarización:

Modelamiento de base de datos

Estudio de Código preexistente y entidades

Se estudiarán el <código> preexistente para determinar sus funcionalidades y su integración o reemplazo dentro de la nueva plataforma.

A partir del modelo actual se determinarán las entidades, sus atributos y relaciones reportado mediante un diagrama de relaciones de entidades (Entity Relationship Diagram - ERD). Cada módulo o <script> utilizado por la plataforma actual será analizado para determinar un diseño que los integre.

Model/View/Controller

Paradigma MVC

  • Desarrollo más rápido

    1. Desarrollo paralelo
    2. Limpio
    3. Más facil integración

  • 1 Entidad ∞ Vistas

    Podemos producir múltiples vistas para una entidad con validaciones POST sin generar nuevas entidades innecesarias.

  • Carga Asincrónica

    Páginas más rápidas, más sensibles y más seguras.

  • Modular

    La modificación de una vista, la generación de nuevas vistas, la generación de nuevos modelos o el cambio de una identidad son independientes.

  • Multiformato

    El mismo controlador o modelo puede ser utilizado para crear páginas en cualquier formato de renderizado
